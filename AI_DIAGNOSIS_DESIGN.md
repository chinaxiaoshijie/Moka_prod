# AI诊断功能设计 v2.0

> **更新时间**: 2026-04-15  
> **位置**: 面试流程详情页面 (`/interview-processes/[id]/page.tsx`)  
> **核心逻辑**: 每一轮面试反馈前增加"AI诊断"，根据前序信息生成诊断结果

---

## 一、功能定位

在面试流程时间线中，**每一轮面试反馈之前**插入"AI诊断"卡片，帮助HR和面试官：
- **HR面试**: 基于简历+岗位要求生成候选人分析和面试建议
- **后续轮次**: 基于前一轮面试官反馈，生成诊断结果和下一轮面试重点

---

## 二、业务流程

```
候选人简历导入
    ↓
HR匹配岗位
    ↓
【第1轮 HR面试】
    ├── AI诊断（基于简历+岗位）
    ├── 面试安排
    ├── HR面试反馈
    └── HR确认下一步
    ↓
【第2轮 技术面试】
    ├── AI诊断（基于HR反馈）
    ├── 面试安排
    ├── 技术面试官反馈
    └── HR确认下一步
    ↓
【第3轮 终面】
    ├── AI诊断（基于技术面反馈）
    ├── 面试安排
    ├── 终面面试官反馈
    └── HR确认录用/拒绝
```

---

## 三、AI诊断触发逻辑

| 轮次 | 触发条件 | 输入数据 | 诊断内容 |
|------|----------|----------|----------|
| **HR面试（第1轮）** | HR匹配岗位后/进入流程详情页 | 候选人简历文本 + 职位要求 | 候选人能力分析、岗位匹配度、HR面试重点建议 |
| **技术面试（第2轮）** | HR确认通过HR轮后 | HR面试反馈（优势/不足/评语） + 职位要求 | 技术面考察重点、需深挖的能力项、风险提示 |
| **终面（第3轮）** | 技术面反馈完成后 | 技术面反馈 + HR反馈 + 职位要求 | 终面综合评估重点、决策建议、风险项 |
| **后续轮次** | 前轮反馈完成后 | 所有前轮反馈汇总 + 职位要求 | 综合诊断、决策建议 |

## 五、数据结构设计

### 5.1 Prisma Schema 扩展

```prisma
// AI诊断结果表
model AIDiagnosis {
  id           String   @id @default(uuid())
  processId    String   // 面试流程ID
  roundNumber  Int      // 针对第几轮的诊断
  positionId   String?  // 关联职位
  
  // 诊断结果
  matchScore   Int?     // 匹配度 0-100（仅HR轮）
  matchLevel   String?  // high/medium/low
  strengths    String[] // 优势
  weaknesses   String[] // 不足/风险
  suggestions  String[] // 面试建议/重点
  questions    String[] // 建议面试问题（后续轮次）
  summary      String   // 前轮反馈摘要（后续轮次）
  rawOutput    String   // AI原始输出
  
  // 输入数据快照（用于重新分析时追溯）
  inputSnapshot Json?   // 记录生成诊断时的输入数据
  
  analyzedAt   DateTime @default(now())
  analyzedBy   String   @default("qwen-plus")
  
  process      InterviewProcess @relation(fields: [processId], references: [id])
  position     Position? @relation(fields: [positionId], references: [id])
  
  @@unique([processId, roundNumber])  // 每轮只有一个诊断
  @@map("ai_diagnoses")
}
```

### 5.2 接口类型定义

```typescript
interface AIDiagnosis {
  id: string;
  processId: string;
  roundNumber: number;
  matchScore: number | null;
  matchLevel: string | null;
  strengths: string[];
  weaknesses: string[];
  suggestions: string[];
  questions: string[];
  summary: string;
  rawOutput: string;
  analyzedAt: string;
}
```

### 8.2 AIDiagnosisCard 组件

```typescript
interface AIDiagnosisCardProps {
```

### 8.3 状态管理

- **诊断状态**: `pending` | `loading` | `completed` | `error`
- **缓存策略**: 同一轮次不重复生成，除非手动点击"重新分析"
- **错误处理**: AI调用失败时显示错误信息，支持重试

## 十一、注意事项

- **AI调用成本**: 每次诊断约消耗 2000-4000 token，单次成本约 ¥0.01
- **缓存策略**: 同一轮次不重复生成，避免浪费
- **错误处理**: AI服务不可用时不应阻塞面试流程
- **权限控制**: 仅HR可触发诊断和分享
- **数据隐私**: 诊断结果仅对相关人员可见
