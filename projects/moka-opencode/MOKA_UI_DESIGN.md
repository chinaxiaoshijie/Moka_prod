## 📊 Moka 核心界面结构分析

### 1. 主导航栏

基于 Moka 官网和功能描述，主导航应包含以下模块：

| 模块 | 描述 | 优先级 |
|------|------|--------|
| **工作台 (Dashboard)** | 招聘漏斗概览、待处理任务、数据统计 | 高 |
| **候选人** | 候选人列表、筛选、搜索 | 高 |
| **职位** | 职位管理、发布状态、招聘进度 | 高 |
| **面试** | 面试日历、面试列表、面试安排 | 高 |
| **人才库** | 候选人储备、标签管理 | 中 |
| **报表** | 招聘数据分析、渠道效果、漏斗分析 | 低 |

### 3. 候选人管理界面

#### 3.1 候选人列表页

**顶部筛选栏**:
```
┌──────────────────────────────────────────────────────────────┐
│ 🔍 搜索: [姓名/电话/邮箱输入框]              │
│ 状态: [全部 ▼] | 职位: [全部 ▼] | 时间: [全部 ▼] │
│ [+ 添加候选人] [+ 批量导入] [+ 导出]                 │
└──────────────────────────────────────────────────────────────┘
```

**候选人列表列**:
| 列 | 字段 | 说明 |
|------|------|------|
| 姓名 | name | 候选人姓名 |
| 职位 | position_name | 申请的职位 |
| 状态 | status | 待处理/初试/复试/终试/已录用/已淘汰 |
| 电话 | phone | 联系电话 |
| 邮箱 | email | 联系邮箱 |
| 来源 | source | BOSS直聘/内推/猎头 |
| 更新时间 | updated_at | 最后更新时间 |
| 操作 | actions | 查看/编辑/@面试官/删除 |

**状态颜色标识**:
- 待处理: 🔵 灰色
- 初试: 🟢 绿色
- 复试: 🟡 黄色
- 终试: 🟠 橙色
- 已录用: ✅ 绿色高亮
- 已淘汰: ❌ 红色

## 🎨 UI 设计规范建议

### 颜色主题

基于 Moka 的现代 UI 风格：

| 用途 | 主色 | 十六进制 | 说明 |
|------|------|----------|------|
| 主题色 | 蓝色 | #0066FF | 主按钮、链接 |
| 成功 | 绿色 | #52C41A | 已录用、通过 |
| 警告 | 黄色 | #FAAD14 | 待定、注意 |
| 危险 | 红色 | #FF4D4F | 已淘汰、错误 |
| 信息 | 浅蓝 | #1890FF | 提示信息 |
| 背景 | 灰白 | #F5F5F5 | 页面背景 |
| 卡片 | 白色 | #FFFFFF | 内容卡片 |

### 字体规范

| 类型 | 字体 | 大小 | 用途 |
|------|------|------|------|
| 标题 | PingFang SC, Microsoft YaHei | 18-24px | 页面标题 |
| 正文 | PingFang SC, Microsoft YaHei | 14px | 正文内容 |
| 辅助 | PingFang SC, Microsoft YaHei | 12px | 辅助说明 |

### 组件规范

#### 按钮

- 主按钮：主题色背景，白色文字，圆角 4px

## 🔄 工作流程细化

### 候选人入职流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 简历    │ →  │ 初步    │ →  │ @面试官 │ →  │ 初试    │
│ 收到    │    │ 筛选    │    │ 查看    │    │ 安排    │
└─────────┘    └─────────┘    └─────────┘    └─────────┘
                                                  ↓
                                            ┌─────────┐
                                            │ 初试    │
                                            │ 反馈    │
                                            └─────────┘
                                                  ↓
                                            ┌─────────┐
                                            │ 是否    │
                                            │ 通过?   │
                                            └─────────┘
                                             /    \
                                           是      否
                                           /        \
                                  ┌─────────┐    ┌─────────┐
                                  │ 复试    │    │ 淘汰    │
                                  │ 安排    │    │ 通知    │
                                  └─────────┘    └─────────┘
                                        ↓
                                  ┌─────────┐
                                  │ 复试    │
                                  │ 反馈    │
                                  └─────────┘
                                        ↓
                                  ┌─────────┐
                                  │ 终试    │
                                  │ 安排    │
                                  └─────────┘
                                        ↓
                                  ┌─────────┐
                                  │ Offer   │
                                  │ 发放    │
                                  └─────────┘
```

### 面试安排流程

```
1. HR 筛选简历
   ↓
2. HR @ 面试官
   ↓
3. 面试官查看简历
   ↓
4. 面试官同意 → 安排初试
   ↓
5. 系统发送通知（候选人 + 面试官）
   ↓
6. 面试进行中
   ↓
7. 系统发送反馈链接给面试官
   ↓
8. 面试官填写反馈
   ↓
9. HR 查看反馈，决定是否进入下一轮
```
