## API端点汇总

### 认证模块

- `POST /auth/login` - 登录
- `GET /auth/profile` - 获取用户信息
- `GET /auth/users` - 获取用户列表

### 职位模块

- `GET /positions` - 职位列表
- `POST /positions` - 创建职位
- `PUT /positions/:id` - 更新职位
- `DELETE /positions/:id` - 删除职位

### 候选人模块

- `GET /candidates` - 候选人列表
- `POST /candidates` - 创建候选人
- `PUT /candidates/:id` - 更新候选人
- `DELETE /candidates/:id` - 删除候选人
- `POST /candidates/parse-resume` - 解析简历

### 面试模块

- `GET /interviews` - 面试列表
- `POST /interviews` - 创建面试
- `GET /interviews/:id` - 面试详情
- `PUT /interviews/:id` - 更新面试
- `DELETE /interviews/:id` - 删除面试

### 面试流程模块

- `GET /interview-processes` - 流程列表
- `POST /interview-processes` - 创建流程
- `GET /interview-processes/:id` - 流程详情
- `POST /interview-processes/:id/complete-round` - 完成轮次

### 反馈模块

- `GET /feedback` - 反馈列表
- `POST /feedback` - 提交反馈
- `GET /feedback/interview/:id` - 面试反馈

### 通知模块

- `GET /notifications` - 通知列表
- `POST /notifications` - 创建通知
- `PUT /notifications/:id/read` - 标记已读
- `PUT /notifications/read-all` - 全部已读
- `DELETE /notifications/:id` - 删除通知

### 数据分析模块

- `GET /analytics/funnel` - 招聘漏斗
- `GET /analytics/interviewers` - 面试官统计
- `GET /analytics/sources` - 来源分析
- `GET /analytics/timeline` - 招聘时间线
- `GET /analytics/hiring-cycle` - 平均招聘周期
- `GET /analytics/dashboard` - 仪表盘统计

## 部署说明

### 环境要求

- Node.js >= 18
- PostgreSQL >= 14
- Bun >= 1.3

### 部署步骤

1. 安装依赖: `bun install`
2. 配置环境变量
3. 数据库迁移: `bunx prisma migrate deploy`
4. 构建: `bun run build`
5. 启动: `bun run start`
