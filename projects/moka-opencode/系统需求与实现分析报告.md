# Moka面试管理系统 - 需求与实现分析报告

## 系统概述

Moka面试管理系统是一个企业级招聘管理解决方案，涵盖从职位发布、候选人管理、面试流程到反馈收集的完整招聘流程。

**技术栈**：

- **后端**: NestJS + Prisma + PostgreSQL + JWT认证
- **前端**: Next.js + React + TypeScript + Tailwind CSS
- **测试**: Bun Test + 自定义测试脚本

---

## 数据模型（Prisma Schema）

系统包含以下核心实体：

### 1. 用户管理 (User)

- **字段**: id, username, password, name, email, role, avatarUrl
- **角色**: HR, INTERVIEWER
- **关联**: 面试、面试流程、反馈

### 2. 职位管理 (Position)

- **字段**: id, title, description, requirements, salaryMin/Max, headcount, status, location
- **状态**: OPEN, PAUSED, CLOSED
- **关联**: 候选人、面试、面试流程

### 3. 候选人管理 (Candidate)

- **字段**: id, name, phone, email, positionId, status, source, resumeUrl
- **状态**: PENDING, SCREENING, INTERVIEW_1, INTERVIEW_2, INTERVIEW_3, HIRED, REJECTED
- **来源**: BOSS, REFERRAL, HEADHUNTER, WEBSITE
- **关联**: 职位、面试、面试流程

### 4. 面试管理 (Interview)

- **字段**: id, candidateId, positionId, interviewerId, type, format, startTime, endTime, location, meetingUrl, status
- **类型**: INTERVIEW_1, INTERVIEW_2, INTERVIEW_3
- **形式**: ONLINE, OFFLINE
- **状态**: SCHEDULED, COMPLETED, CANCELLED

### 5. 面试流程 (InterviewProcess)

- **字段**: id, candidateId, positionId, currentRound, totalRounds, status, hasHRRound
- **状态**: IN_PROGRESS, WAITING_HR, COMPLETED, CANCELLED
- **关联**: 候选人、职位、面试轮次、面试

### 6. 面试反馈 (InterviewFeedback)

- **字段**: id, interviewId, interviewerId, result, strengths, weaknesses, overallRating, notes
- **结果**: PASS, FAIL, PENDING
- **评分**: 1-5星

## 缺失功能清单

### 🔴 高优先级（核心功能缺失）

#### 1. 面试日历/日程管理

**需求**：

- 以日历形式展示面试安排
- 支持日/周/月视图
- 面试提醒功能
- 与面试官日程冲突检测

**状态**：❌ 未实现

#### 2. 完整的面试安排流程

**需求**：

- 为面试流程的每一轮安排具体面试时间
- 选择面试形式（线上/线下）
- 设置会议室或会议链接
- 发送面试邀请（邮件/系统通知）

**状态**：⚠️ 部分实现

#### 3. 面试反馈完整流程

**需求**：

- 面试官在面试后提交反馈表单
- 评分系统（1-5星）
- 通过/不通过决策
- 综合多轮面试反馈生成最终报告

**状态**：⚠️ 基础API存在，前端待完善

#### 4. 候选人状态流转

**需求**：

- 清晰的候选人状态机
- 自动状态更新（如初试通过→复试）
- 状态变更历史记录

**状态**：⚠️ 状态字段存在，流转逻辑待完善

### 🟡 中优先级（用户体验优化）

#### 5. 通知系统

#### 6. 文件上传与管理

#### 7. 数据导入导出

#### 8. 搜索与筛选增强
