## 项目概述

### 目标
构建一个仿照 Moka ATS 的招聘面试管理系统，提供候选人管理、面试安排、日程同步和面试反馈等核心功能。

### 背景
用户当前使用 Moka ATS 系统进行招聘管理，主要流程包括职位发布、简历导入、候选人管理、面试安排和面试反馈等。目标是构建一个独立系统来替代这些功能。

## 用户原始需求

### 当前 Moka 使用流程

1. **职位发布**
   - 在 Moka 发布新职位
   - 一键发布到 BOSS 直聘

2. **简历导入**
   - BOSS 直聘收到的简历直接导入到 Moka
   - 自动查重功能

3. **核心功能**
   - 候选人管理
   - 面试安排
   - 职位管理

4. **简历评审**
   - 在 Moka 中将合适简历 @ 面试官
   - 面试官查看简历是否合适
   - 合适的进入面试安排环节

5. **面试流程**
   - 分为初试、复试和终试
   - 支持线上视频面试和现场面试

6. **线上面试**
   - 使用腾讯会议
   - 发送面试链接、时间等信息给候选人和面试官
   - 日程同步到飞书日历

7. **线下面试**
   - 发送地点、时间等信息给候选人和面试官
   - 日程同步到飞书日历

8. **面试反馈**
   - 同步发送面试反馈链接给面试官
   - 面试后填写反馈

9. **反馈内容**
   - 评价结果：满意、不满意、待定
   - 综合评价：从优势和劣势对候选人分别评价

1. **需求确认**
   - 登录 Moka 网站明确功能和需求

6. **平台要求**

## 非功能需求

### 性能要求
- 系统响应时间：页面加载 < 2 秒
- 支持并发用户：至少 20 人同时使用
- 文件上传：支持最大 10MB 简历文件

### 可用性要求
- 系统可用性：99%（工作时间内）
- 界面友好，易于操作

### 安全要求
- 用户身份验证
- 数据传输加密（HTTPS）
- 敏感信息保护（密码、手机号）

### 兼容性要求
- 支持主流浏览器（Chrome、Firefox、Edge）
- PC 端响应式设计

## 数据模型

### 1. 用户表 (User)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| name | VARCHAR(100) | 姓名 |
| email | VARCHAR(100) | 邮箱（登录用） |
| phone | VARCHAR(20) | 手机号 |
| password | VARCHAR(255) | 密码（加密） |
| role | VARCHAR(20) | 角色（admin/hr/interviewer） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 2. 职位表 (Position)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| title | VARCHAR(200) | 职位名称 |
| description | TEXT | 职位描述 |
| requirements | TEXT | 职位要求 |
| salary_range | VARCHAR(50) | 薪资范围 |
| headcount | INT | 招聘人数 |
| status | VARCHAR(20) | 状态（active/paused/closed） |
| created_by | INT | 创建人 ID |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 3. 候选人表 (Candidate)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| name | VARCHAR(100) | 姓名 |
| phone | VARCHAR(20) | 手机号 |
| email | VARCHAR(100) | 邮箱 |
| resume_url | VARCHAR(500) | 简历文件路径 |
| position_id | INT | 申请职位 ID |
| status | VARCHAR(20) | 状态（pending/interview_1/interview_2/interview_3/hired/rejected） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 4. 面试表 (Interview)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| candidate_id | INT | 候选人 ID |
| position_id | INT | 职位 ID |
| interview_type | VARCHAR(20) | 面试类型（first/second/final） |
| interview_mode | VARCHAR(20) | 面试形式（online/offline） |
| start_time | DATETIME | 开始时间 |
| end_time | DATETIME | 结束时间 |
| location | VARCHAR(200) | 面试地点（线下） |
| meeting_url | VARCHAR(500) | 会议链接（线上） |
| meeting_id | VARCHAR(100) | 会议号（线上） |
| interviewers | JSON | 面试官列表 [id1, id2, ...] |
| status | VARCHAR(20) | 状态（scheduled/completed/cancelled） |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

### 5. 面试反馈表 (InterviewFeedback)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT | 主键 |
| interview_id | INT | 面试 ID |
| interviewer_id | INT | 面试官 ID |
| result | VARCHAR(20) | 评价结果（satisfied/unsatisfied/pending） |
| strengths | TEXT | 优势评价 |
| weaknesses | TEXT | 劣势评价 |
| comments | TEXT | 综合评价 |
| submitted_at | DATETIME | 提交时间 |

## 用户故事

### 候选人管理

**作为 HR，**
我想要添加候选人信息并上传简历，
以便管理候选人。

**作为 HR，**
我想要搜索和筛选候选人，
以便快速找到特定状态的候选人。

**作为 HR，**
我想要将候选人简历 @ 给面试官，
以便面试官可以查看并决定是否安排面试。

### 职位管理

**作为 HR，**
我想要创建和编辑职位信息，
以便发布招聘需求。

**作为 HR，**
我想要导出职位为发布格式，
以便人工发布到 BOSS 直聘。

### 面试安排

**作为 HR，**
我想要安排线上面试（腾讯会议），
以便候选人可以远程参加面试。

**作为 HR，**
我想要安排线下面试，
以便候选人到公司参加面试。

**作为 HR，**
我想要查看面试日历，
以便了解所有面试安排。

### 面试反馈

**作为面试官，**
我想要收到面试反馈链接，
以便在面试后填写反馈。

**作为面试官，**
我想要填写候选人优势和劣势评价，
以便记录面试结果。

**作为 HR，**
我想要查看所有面试反馈汇总，
以便做录用决策。

### 通知和集成

**作为候选人，**
我想要收到面试通知邮件，
以便了解面试时间、地点和链接。

**作为面试官，**
我想要收到面试通知和反馈提醒，
以便准时参加面试并及时填写反馈。

**作为 HR，**
我想要导出面试日历事件（ICS），
以便手动同步到飞书日历。
