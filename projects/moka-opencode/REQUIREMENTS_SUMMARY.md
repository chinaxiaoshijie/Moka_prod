# 需求总结和分析

**日期**: 2026-02-12
**分析人**: System

---

## 📊 需求概览

### 项目目标
构建一个替代 Moka ATS 的招聘面试管理系统，核心功能包括候选人管理、面试安排、日程同步和面试反馈。

### 核心价值
- 简化招聘流程
- 提高面试安排效率
- 集成飞书日历（主要工作工具）
- 支持线上和线下面试
- 自动化邮件通知

---

## 🎯 关键功能模块

### 1. 候选人管理（优先级：高）
**核心需求**:
- 候选人 CRUD 操作
- 简历上传和预览
- 简历去重（姓名+电话）
- 候选人筛选和搜索
- @面试官查看简历

**数据量**: 中等
**复杂度**: 中等

---

### 2. 职位管理（优先级：高）
**核心需求**:
- 职位 CRUD 操作
- 导出职位信息（供人工发布到 BOSS 直聘）

**数据量**: 小
**复杂度**: 低

---

### 3. 面试安排（优先级：高）
**核心需求**:
- 面试类型：初试、复试、终试
- 面试形式：线上（腾讯会议）、线下（现场）
- 线上面试：记录腾讯会议链接和会议号
- 线下面试：记录面试地点
- 面试日历视图

**数据量**: 中等
**复杂度**: 高（时间安排、日历视图）

---

### 4. 面试反馈（优先级：高）
**核心需求**:
- 发送反馈链接给面试官
- 面试官在线填写反馈
- 评价结果：满意、不满意、待定
- 优势/劣势评价
- 反馈汇总和导出

**数据量**: 中等
**复杂度**: 中等

---

### 5. 邮件通知（优先级：中）
**核心需求**:
- 面试通知给候选人（时间、地点/链接）
- 面试通知给面试官（简历、时间、反馈链接）
- 反馈填写提醒
- 邮件模板管理

**数据量**: 大（每个操作都可能触发）
**复杂度**: 中等

---

### 6. 飞书日历集成（优先级：中）
**核心需求（第一阶段）**:
- 生成 ICS 日历事件文件
- 手动导入到飞书日历

**未来需求**:
- API 自动同步日程
- 单点登录（SSO）

**数据量**: 小
**复杂度**: 中等（第一阶段）/ 高（API 集成）

---

### 7. 腾讯会议集成（优先级：低）
**核心需求（第一阶段）**:
- 手动创建腾讯会议
- 记录会议链接和会议号

**未来需求**:
- API 自动创建腾讯会议

**数据量**: 小
**复杂度**: 低（第一阶段）/ 中等（API 集成）

---

### 8. BOSS 直聘对接（优先级：低）
**核心需求（第一阶段）**:
- 人工导入简历（提供标准格式）
- 人工导出职位（提供标准格式）

**未来需求**:
- API 自动对接

**数据量**: 大（简历导入）
**复杂度**: 低（人工处理）/ 高（API 对接）

---

## 🛠️ 技术选型建议

### 前端框架选项

| 框架 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| React + Ant Design | 生态成熟，组件丰富 | 首次加载慢 | ⭐⭐⭐⭐⭐ |
| Vue 3 + Element Plus | 学习曲线低，性能好 | 生态相对小 | ⭐⭐⭐⭐ |
| Next.js + shadcn/ui | 现代，SSR 支持 | 需要更多配置 | ⭐⭐⭐⭐⭐ |

**推荐**: Next.js + shadcn/ui 或 React + Ant Design

---

### 后端框架选项

| 框架 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| Express + TypeScript | 简单，灵活 | 需要手动配置 | ⭐⭐⭐⭐ |
| NestJS | 结构清晰，内置依赖注入 | 学习曲线较陡 | ⭐⭐⭐⭐⭐ |
| Fastify + TypeScript | 高性能，TypeScript 原生 | 生态相对小 | ⭐⭐⭐⭐ |
| Go + Gin | 高性能，部署简单 | TypeScript 支持差 | ⭐⭐⭐ |

**推荐**: NestJS（结构化）或 Express（简单快速）

---

### 数据库选项

| 数据库 | 优势 | 劣势 | 推荐度 |
|--------|------|------|--------|
| PostgreSQL | 功能强大，JSON 支持 | 配置相对复杂 | ⭐⭐⭐⭐⭐ |
| MySQL | 成熟稳定，易用 | JSON 支持弱 | ⭐⭐⭐⭐ |
| SQLite | 零配置，适合 MVP | 并发能力弱 | ⭐⭐⭐ |

**推荐**: PostgreSQL（生产）/ SQLite（MVP）

## ⚠️ 风险和挑战

### 技术风险

1. **日历时间冲突检测**
   - 挑战：面试官可能同时安排多个面试
   - 缓解：时间冲突检测逻辑，提前提示

2. **邮件发送可靠性**
   - 挑战：邮件可能被误判为垃圾邮件
   - 缓解：使用企业邮箱，配置 SPF/DKIM

3. **文件上传和存储**
   - 挑战：简历文件大小不一，格式多样
   - 缓解：限制文件大小，支持常见格式

### 业务风险

1. **飞书集成复杂性**
   - 挑战：飞书 API 可能有变更
   - 缓解：第一阶段用 ICS 导入，后续再 API 集成

2. **用户培训成本**
   - 挑战：从 Moka 迁移需要培训
   - 缓解：界面友好，提供操作文档

3. **数据迁移**
   - 挑战：如果需要迁移 Moka 数据
   - 缓解：设计数据导入工具
