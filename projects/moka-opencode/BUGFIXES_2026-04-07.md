## 📋 修复内容

### 1. 竞态条件修复（使用 upsert 替代 findFirst + update/create）

**问题**: 高并发下可能重复创建同一轮次的面试

**修复文件**:
- `backend/src/interviews/interview.service.ts` - `create()` 方法
- `backend/src/interview-processes/interview-process.service.ts` - `createInterviewForRound()` 方法

**修复方案**: 使用 Prisma 的 `upsert` 方法，基于复合唯一键 `processId_roundNumber` 确保同一流程的同一轮次只有一个面试记录

**代码变更**:
```typescript
// 修复前
const existingInterview = await this.prisma.interview.findFirst({
  where: { processId, roundNumber }
});
if (existingInterview) {
  return await this.prisma.interview.update(...);
} else {
  return await this.prisma.interview.create(...);
}

// 修复后
const interview = await this.prisma.interview.upsert({
  where: {
    processId_roundNumber: { processId, roundNumber }
  },
  create: { ... },
  update: { ... },
});
```

### 2. 邮件重复发送防护

**问题**: HR 可能误操作重复发送邮件给候选人

**修复文件**:
- `backend/src/interviews/interview.service.ts` - `sendCandidateEmail()` 方法

**修复方案**:
1. **24 小时限制**: 检查 `InterviewEmailLog` 表，24 小时内同一面试只能发送一次
2. **发送日志记录**: 成功和失败的邮件都记录到数据库
3. **友好错误提示**: 告知用户上次发送时间

**代码变更**:
```typescript
// 检查 24 小时内是否已发送过邮件
const twentyFourHoursAgo = new Date(Date.now() - 24 * 60 * 60 * 1000);
const recentEmailLogs = await this.prisma.interviewEmailLog.findMany({
  where: {
    interviewId: interviewId,
    sentAt: { gte: twentyFourHoursAgo },
  },
  orderBy: { sentAt: "desc" },
  take: 1,
});

if (recentEmailLogs.length > 0) {
  const lastSent = recentEmailLogs[0].sentAt;
  throw new Error(
    `24 小时内已发送过邮件，请勿重复发送（上次发送时间：${lastSent.toLocaleString("zh-CN", { timeZone: "Asia/Shanghai" })}）`
  );
}

// 记录邮件发送日志
await this.prisma.interviewEmailLog.create({
  data: {
    interviewId,
    candidateId,
    recipientEmail,
    subject,
    content,
    sentBy,
    status: "sent" | "failed",
    errorMessage,
  },
});
```

## 📝 注意事项

1. **唯一键约束**: 如果数据库中已存在重复数据，迁移会失败。需要先清理：
   ```sql
   -- 查找重复数据
   SELECT "processId", "roundNumber", COUNT(*) 
   FROM interviews 
   GROUP BY "processId", "roundNumber" 
   HAVING COUNT(*) > 1;
   
   -- 删除重复数据（保留最新的）
   DELETE FROM interviews a USING interviews b
   WHERE a."processId" = b."processId" 
     AND a."roundNumber" = b."roundNumber" 
     AND a."createdAt" < b."createdAt";
   ```

2. **邮件日志**: 现有邮件发送不会显示在日志中（因为之前没有记录），新功能上线后的发送才会被记录

3. **24 小时限制**: 如果需要调整时间窗口，修改 `sendCandidateEmail()` 方法中的 `24 * 60 * 60 * 1000`
