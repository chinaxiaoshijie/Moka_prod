## 修复内容

### 更新候选人功能修复 ✅

**问题**: 测试脚本中更新候选人时使用固定电话号码，导致数据库唯一约束冲突

**根本原因**:

- 数据库中 `Candidate` 表有唯一约束 `@@unique([name, phone])`
- 测试脚本在更新时使用了固定的电话号码 `13800138001`
- 与数据库中已有数据冲突，导致返回404错误

**修复方案**:

```typescript
// 修复前：使用固定电话号码
const response = await this.request("PUT", `/candidates/${this.candidateId}`, {
  name: "测试候选人张三（已更新）",
  phone: "13800138001",  // ❌ 固定值，可能冲突
  email: "zhangsan_updated@test.com",
}, this.hrToken);

// 修复后：先获取当前候选人信息，使用相同的电话号码
const getResponse = await this.request("GET", `/candidates/${this.candidateId}`, ...);
const currentPhone = getResponse.body.phone;

const response = await this.request("PUT", `/candidates/${this.candidateId}`, {
  name: "测试候选人张三（已更新）",
  phone: currentPhone,  // ✅ 使用原电话号码，避免冲突
  email: "zhangsan_updated@test.com",
}, this.hrToken);
```

**验证结果**: ✅ 修复后测试通过
